Exploratory Data Analysis (EDA) is the process of investigating a dataset to understand its structure, spot patterns, find anomalies, and form hypotheses before building any model. Every professional data scientist runs a full EDA before touching a model. This project puts everything from Unit 3 together into one realistic pipeline using a real-world dataset.

Project Overview
  • Load and inspect a real dataset using Pandas.
  • Clean the data: handle missing values, fix types, remove duplicates.
  • Compute descriptive statistics and group summaries.
  • Answer five specific analytical questions about the dataset.
  • Export a clean, analysis-ready CSV file.

The Dataset

Use the Titanic dataset — one of the most famous beginner datasets in data science. Download it from the link below and save it as titanic.csv in your project folder.

ℹ️ Download the Titanic dataset from: Titanic CSV — datasciencedojo/datasets. Save it as titanic.csv in the same folder as your script.

Key columns you will work with:

  • Survived — 0 = did not survive, 1 = survived
  • Pclass — passenger class: 1 = first, 2 = second, 3 = third
  • Name — passenger name
  • Sex — male or female
  • Age — age in years (has missing values)
  • Fare — ticket price
  • Embarked — port of embarkation: C, Q, or S

Requirements

  • All code lives in a single eda.py file.
  • Each section is separated by a clear printed heading.
  • The script runs end-to-end without errors using python3 eda.py.
  • A cleaned version of the dataset is saved as titanic_clean.csv.

Hints

  • Survival rate: df["Survived"].mean() gives the proportion of survivors.
  • Survival rate by group: df.groupby("Sex")["Survived"].mean()
  • Formatting as percentage: multiply by 100 and round to 1 decimal.
  • Top 5 fares: df["Fare"].nlargest(5)
  • Class labels: map Pclass values 1, 2, 3 to "1st", "2nd", "3rd" using .map() before printing.
