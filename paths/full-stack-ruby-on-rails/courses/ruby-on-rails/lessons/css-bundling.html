Modern Rails apps use CSS Bundling or Tailwind CSS instead of (or alongside) the traditional asset pipeline. This lesson covers the options available in Rails 7.

Lesson Overview
  • Understand the difference between Sprockets, Propshaft, and jsbundling.
  • Set up Tailwind CSS in a Rails 7 app.
  • Set up cssbundling-rails with PostCSS.
  • Understand importmaps for JavaScript.

Rails 7 CSS Options

  • Sprockets (classic) — the traditional asset pipeline. Still works, widely understood.
  • Propshaft — modern, simpler replacement for Sprockets. No compilation, just fingerprinting.
  • cssbundling-rails — delegates to an external bundler: PostCSS, Tailwind, Bootstrap, Bulma.
  • importmap-rails — uses browser ES modules for JavaScript, no bundler needed.
# Create a new Rails 7 app with Tailwind
rails new myapp --css tailwind --database=postgresql

# Or add Tailwind to existing app
bundle add tailwindcss-rails
rails tailwindcss:install

# Start both Rails server and CSS watcher
bin/dev   # runs Procfile.dev (foreman/overmind)

/* app/assets/stylesheets/application.tailwind.css */
@tailwind base;
@tailwind components;
@tailwind utilities;

/* Custom components */
@layer components {
  .btn-primary {
    @apply bg-blue-600 text-white px-4 py-2 rounded hover:bg-blue-700;
  }
}
