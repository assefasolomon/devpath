Rails can serve as both a traditional server-rendered app and a JSON API. This lesson covers how to build a REST API with Rails and how to consume external APIs.

Lesson Overview
  • Render JSON responses from controllers.
  • Use jbuilder or ActiveModelSerializers.
  • Handle authentication for API endpoints.
  • Make HTTP requests to external APIs with Faraday.

JSON Responses

class Api::V1::PostsController < ApplicationController
  before_action :authenticate_api_user!

  def index
    @posts = Post.published.includes(:user).page(params[:page])
    render json: @posts, each_serializer: PostSerializer
  end

  def show
    @post = Post.find(params[:id])
    render json: @post, serializer: PostSerializer
  end

  def create
    @post = current_user.posts.build(post_params)
    if @post.save
      render json: @post, status: :created
    else
      render json: { errors: @post.errors.full_messages }, status: :unprocessable_entity
    end
  end
end

# config/routes.rb
namespace :api do
  namespace :v1 do
    resources :posts, only: [:index, :show, :create, :update, :destroy]
    resources :users, only: [:show, :create]
  end
end

Consuming External APIs

# Gemfile
gem 'faraday'

# app/services/weather_service.rb
class WeatherService
  BASE_URL = "https://api.openweathermap.org/data/2.5"

  def self.current(city)
    response = Faraday.get("#{BASE_URL}/weather") do |req|
      req.params[:q]     = city
      req.params[:appid] = ENV['OPENWEATHER_API_KEY']
      req.params[:units] = 'metric'
    end
    JSON.parse(response.body)
  end
end

# Use in controller
@weather = WeatherService.current("London")
