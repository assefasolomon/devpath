CSS frameworks like Bootstrap and Tailwind, and preprocessors like Sass, extend what plain CSS can do. This lesson covers when and why to use them, and how they integrate with Rails.

Lesson Overview
  • Understand the difference between utility-first and component-based CSS frameworks.
  • Know when to use Bootstrap vs Tailwind.
  • Understand what Sass adds to CSS.
  • Know how Rails integrates CSS frameworks.

CSS Frameworks

# Bootstrap — component-based, opinionated design system
# Install in Rails:
bundle add bootstrap
# Gives you: pre-styled buttons, cards, modals, navbars, grid
# Pro: fast prototyping, familiar to many devs
# Con: sites look alike, hard to customise deeply

# Tailwind CSS — utility-first
# Install in Rails 7:
rails new myapp --css tailwind
# Gives you: single-purpose utility classes
# Pro: fully customisable, no specificity wars, great for custom designs
# Con: verbose HTML, learning curve

<!-- Bootstrap -->
<button class="btn btn-primary">Save</button>

<!-- Tailwind -->
<button class="bg-blue-600 text-white px-4 py-2 rounded hover:bg-blue-700">Save</button>

Sass/SCSS

// Sass adds: variables, nesting, mixins, functions, partials

// Variables (use CSS custom properties instead in modern projects)
$color-primary: #3b82f6;

// Nesting
.card {
  padding: 1rem;
  &:hover { box-shadow: 0 4px 12px rgba(0,0,0,.1); }
  &__title { font-size: 1.25rem; }   // BEM with &
}

// Mixin
@mixin flex-center {
  display: flex;
  align-items: center;
  justify-content: center;
}
.hero { @include flex-center; }

// Rails includes Sass via sassc-rails or dartsass-rails
gem 'dartsass-rails'
