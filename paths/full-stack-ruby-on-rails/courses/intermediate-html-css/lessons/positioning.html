CSS positioning — static, relative, absolute, fixed, and sticky — controls exactly where elements appear. Getting positioning right is essential for modals, dropdowns, tooltips, and sticky headers.

Lesson Overview
  • Understand all five position values.
  • Use absolute positioning relative to a positioned parent.
  • Build a sticky header and a fixed overlay.
  • Use z-index correctly.

Position Values

/* static (default) — normal document flow */
.box { position: static; }

/* relative — offset from normal position, creates stacking context */
.box { position: relative; top: 10px; left: 20px; }

/* absolute — removed from flow, positioned relative to nearest positioned ancestor */
.parent { position: relative; }
.badge  { position: absolute; top: -8px; right: -8px; }

/* fixed — relative to viewport, stays on scroll */
.navbar { position: fixed; top: 0; left: 0; right: 0; z-index: 100; }

/* sticky — hybrid: relative until scroll threshold, then fixed */
.table-header { position: sticky; top: 0; background: white; }

z-index and Stacking Contexts

/* z-index only works on positioned elements (not static) */
.modal-overlay { position: fixed; z-index: 200; }
.modal         { position: fixed; z-index: 201; }
.navbar        { position: fixed; z-index: 100; }

/* A new stacking context is created by:
   - position + z-index other than auto
   - opacity < 1
   - transform
   - will-change */
