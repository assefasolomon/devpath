SQL (Structured Query Language) is the universal language of relational databases. This lesson covers the core SQL commands you need to store, retrieve, and manipulate data.

Lesson Overview
  • Understand relational database concepts: tables, rows, columns, primary keys.
  • Write SELECT queries with WHERE, ORDER BY, and LIMIT.
  • Join tables with INNER JOIN and LEFT JOIN.
  • Insert, update, and delete data.

SELECT Queries

-- Select all columns
SELECT * FROM users;

-- Select specific columns
SELECT name, email, created_at FROM users;

-- Filter with WHERE
SELECT * FROM users WHERE age > 25;
SELECT * FROM users WHERE role = 'admin' AND active = true;
SELECT * FROM users WHERE name LIKE 'Al%';  -- starts with 'Al'

-- Sort results
SELECT * FROM users ORDER BY created_at DESC;
SELECT * FROM users ORDER BY name ASC;

-- Limit and offset (pagination)
SELECT * FROM posts ORDER BY created_at DESC LIMIT 10 OFFSET 20;
-- page 3 of 10-per-page results

Joins

-- Tables: users(id, name), posts(id, title, user_id)

-- INNER JOIN — only rows that match in both tables
SELECT users.name, posts.title
FROM posts
INNER JOIN users ON posts.user_id = users.id;

-- LEFT JOIN — all posts, even if user is deleted
SELECT posts.title, users.name
FROM posts
LEFT JOIN users ON posts.user_id = users.id;

-- Multiple joins
SELECT posts.title, users.name, categories.name AS category
FROM posts
JOIN users      ON posts.user_id     = users.id
JOIN categories ON posts.category_id = categories.id
WHERE posts.published = true
ORDER BY posts.created_at DESC;

INSERT, UPDATE, DELETE

-- Insert
INSERT INTO users (name, email, password_hash)
VALUES ('Alice', 'alice@example.com', '$2a$10$...');

-- Update
UPDATE users SET name = 'Alicia' WHERE id = 42;

-- Delete
DELETE FROM users WHERE id = 42;
DELETE FROM posts WHERE created_at < NOW() - INTERVAL '1 year';
