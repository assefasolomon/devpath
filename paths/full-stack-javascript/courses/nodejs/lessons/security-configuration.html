A web application that ignores security is a liability. This lesson covers the essential security measures every Express application needs: environment variables, CORS, rate limiting, and security headers.

Lesson Overview
  • Use environment variables for secrets with dotenv.
  • Set security headers with Helmet.
  • Implement rate limiting to prevent abuse.
  • Configure CORS correctly.
  • Prevent common vulnerabilities: XSS, CSRF, injection.

Environment Variables

npm install dotenv

// .env — NEVER commit this file to Git
MONGODB_URI=mongodb+srv://user:password@cluster.mongodb.net/mydb
SESSION_SECRET=a-very-long-random-string-here
JWT_SECRET=another-random-string
PORT=3000
NODE_ENV=development

// .gitignore
.env
node_modules/

// Load at the top of your entry file
require('dotenv').config();

// Access anywhere
const port = process.env.PORT || 3000;
const mongoUri = process.env.MONGODB_URI;

Helmet — Security Headers

npm install helmet

const helmet = require('helmet');
app.use(helmet());

// Helmet sets these headers automatically:
// X-Content-Type-Options: nosniff
// X-Frame-Options: DENY
// X-XSS-Protection: 1; mode=block
// Content-Security-Policy: ...
// Referrer-Policy: no-referrer
// Strict-Transport-Security: max-age=15552000; includeSubDomains

Rate Limiting

npm install express-rate-limit

const rateLimit = require('express-rate-limit');

// Apply to all requests
app.use(rateLimit({
  windowMs: 15 * 60 * 1000,  // 15 minutes
  max: 100,                   // limit each IP to 100 requests per window
  message: 'Too many requests, please try again later.',
}));

// Stricter limit for auth routes
const authLimiter = rateLimit({
  windowMs: 60 * 60 * 1000,  // 1 hour
  max: 10,                    // 10 login attempts per hour
});
app.use('/login', authLimiter);
app.use('/register', authLimiter);
